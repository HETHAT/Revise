You are helping build a question bank for a student quiz app.
The app supports filtering by: subject, topic, difficulty, and question type.

Given this file inventory, for each file:

1. Decide if it needs chunking:
   - density "dense" OR estimated_pages > 40 → split into logical concept chunks
   - each chunk = one coherent concept group (not a page range)
   - name chunks as: "FILENAME — Part: [CONCEPT GROUP]"
   - aim for 3-5 chunks per large file

2. For each file (or chunk), decide:
   - Best subset of question types from this list:
       mcq         → concept recognition, comparisons, common mistakes
                     exactly 4 choices, no A/B/C prefix, answer is exact text of one choice
       true_false  → misconceptions, subtle distinctions
                     always include explanation
       short_answer → reasoning, explanations, 1-2 sentence answer
       fill_blank  → formulas, syntax, key terms, answer is the missing word/phrase
       comprehension → reading a paragraph/diagram/table then answering,
                       include the passage in the question field
       debug       → spot the error in code or logic,
                     include the buggy code in the question field
       coding      → write a function or algorithm from scratch
       trace       → trace code execution step by step,
                     include the code in the question field

     Pick based on content:
       theory-heavy     → mcq, true_false, short_answer
       algorithm/logic  → trace, debug, short_answer
       programming      → coding, debug, trace, fill_blank
       mixed            → use judgment

   - Recommended quantity:
       light  → 10–15 questions
       medium → 15–20 questions
       dense / chunk → 20–30 questions

   - High priority concepts (most exam-relevant from key_concepts)
   - Low priority concepts (history, anecdotes, secondary details)
   - Difficulty distribution (adjust to domain complexity, default: 30% easy, 50% medium, 20% hard)
   - Type mix percentages
